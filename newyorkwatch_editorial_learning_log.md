# 뉴욕워치 편집 학습 로그

이 파일은 일반 대화 기록장이 아니다.

사용자가 명시적으로 저장하라고 한 반복 원칙, 확정 기준, 실패 패턴만 남긴다. 임시 제목 후보, 1회성 원고, 순간 판단은 저장하지 않는다.

## 저장 정책

- 저장은 사용자가 명시적으로 요청한 경우에만 한다.
- 저장 대상은 다음으로 제한한다.
  - 반복 적용할 글쓰기 원칙
  - 확정된 브랜드/사업 사실
  - 금지 표현과 권장 표현
  - CTA 고정 규칙
  - 이미지 생성 실패 패턴과 고정 스타일
  - 제목 검증 원칙
- 저장하지 않는 것:
  - 임시 제목 후보
  - 완성본 원고
  - 단발성 주제 판단
  - 사용자가 저장하라고 하지 않은 대화 내용

## 사용자 확정 원칙

- 하네스에는 핵심만 남긴다.
- 사용자가 저장하라고 한 것만 업데이트한다.
- 이전 주제를 다음 글에 억지로 연결하지 않는다.
- 연예인 기사성 글은 예물, 매입, 계급도 흐름과 강제로 연결하지 않는다.
- 사용자가 금지한 브랜드나 키워드는 제목과 본문에 넣지 않는다.

## 글쓰기 확정 기준

- 본문은 사진 설명, 이미지 프롬프트, CTA 설명을 제외하고 2500~3000자.
- 완성본 본문은 3000자를 넘기지 않는다.
- 첫 300자 안에 결론을 넣는다.
- AI 브리핑은 직접 통제할 수 없으므로, 출처로 선택되기 쉬운 검색 의도 대응 문서 구조를 만든다.
- 글은 광고문보다 출처형 설명문에 가깝게 쓴다.
- C-Rank 관점에서는 블로그 전체 주제를 명품시계 매입, 시세, 감정, 거래 절차 중심으로 집중한다.
- D.I.A. 관점에서는 개별 글에 주제 적합도, 실제 경험, 정보 충실성, 문서 의도, 독창성, 적시성을 반영한다.
- 핵심 의심이 있는 구간은 질문형 소제목을 우선 사용하되, 모든 소제목을 질문형으로 강제하지 않는다.
- 매입 후기는 결과보다 감정 기준, 가격 변동 이유, 거래 절차를 설명한다.
- 정보는 본문, 표 이미지, 사진 설명, CTA로 분산한다.
- 매입후기와 실사 기반 포토형 글은 사진 → 핵심 텍스트 → 사진 → 핵심 텍스트 구조로 읽기 쉽게 만든다. 글자 수를 무조건 줄이는 것이 아니라 이미지와 본문을 조합해 정보 흐름을 만든다.
- 가격 비교, 감정 기준, 감가 사유, 최종 매입 결과처럼 설득이 필요한 구간은 몇 줄이든 힘 있게 쓴다. 약한 구간만 짧게 넘긴다.
- 시작 멘트는 반복하지 않는다.
- 감성 홍보보다 감정사가 실제로 말하는 직관적 설명을 우선한다.
- 가격과 시세는 고정 비율로 단정하지 않고 모델, 상태, 구성품, 국내외 수요, 재판매 가능성 기준으로 설명한다.

## 이미지 확정 기준

- 사용자가 보내는 실사 사진은 그대로 쓰는 원본 이미지가 아니라 모델, 색상, 상태, 감정 포인트, 사용감, 버클/브레이슬릿 디테일을 읽는 참고 자료다.
- 실사 사진을 참고해 블로그용 이미지를 새로 설계한다. 목표는 `실사감 있는 핵심 장면 + 짧고 큰 이미지 안 텍스트 + 본문 핵심 설명`이다.
- 이미지 생성 AI는 실제 상담/감정 상황처럼 보이는 핵심 이미지를 만든다. 카카오톡이나 문자 상담 이미지는 고객이 사진을 보내고 뉴욕워치가 1차 견적, 방문 안내, 입금 완료를 답하는 맥락이 보여야 한다.
- 이미지 안 텍스트는 한 이미지에 하나의 짧은 핵심 문구만 넣는다. 긴 설명, 표, 복잡한 가격표, 작은 UI 텍스트는 넣지 않는다.
- 앞으로 원고를 줄 때 각 이미지마다 `이미지 안 텍스트`, `생성 프롬프트`, `사진 설명/대체텍스트`, `본문에서 맡는 역할`을 함께 제공한다.
- 중국식 저가 광고 느낌, 어두운 블랙/골드/브라운 전체톤, 골드 그라데이션 타이포는 금지한다.
- 깨지는 한글, 궁서체, 굴림체, 랜덤 SVG 이모지, 과한 아이콘은 금지한다.
- 화이트/블루 앱 템플릿만 반복하는 것도 실패다. 실제 시계, 감정사 손, 쇼케이스, 상담 장면, 제품 디테일을 섞는다.
- 긴 표, 전화번호, 복잡한 가격표, 긴 문장은 AI 생성 이미지에 넣지 않는다. 단, 사용자가 확정한 짧은 CTA 문구나 핵심 문구는 이미지 안에 자연스럽게 포함한다.
- 이미지 프롬프트는 기본 한국어로 제공한다.

### 2026-05-28 이미지 실패 학습

- 실패 사례: AI 생성 단계에서 한글, 전화번호, CTA, 작은 UI, 표 느낌을 모두 넣어 중국식 저가 광고처럼 보인 이미지.
- 수정 기준: AI에는 핵심 장면과 짧고 큰 한글 문구만 맡긴다. 한 이미지에 여러 문장, 작은 UI 텍스트, 복잡한 표를 넣지 않는다.
- CTA 3장은 문구를 고정하되 배경 장면과 구도는 글 주제에 맞게 다양화한다.
- 실사 참고 사진을 그대로 쓰는 것이 목적이 아니라, 실사 사진의 감정 포인트를 읽어 본문과 이미지 설계에 반영하는 것이 목적이다.

## CTA 확정 기준

- 하단 CTA는 3개로 고정한다.
- 1번 CTA 고정 문구: `010-6293-6668 클릭 하시면 전화연결됩니다.`
- 2번 CTA 고정 문구: `매입절차 및 최고가 약속`
- 3번 CTA 고정 문구: `지금 사진보내고 가격확인`
- CTA 이미지는 글 주제에 맞춰 다양하게 바꾸되, 위 문구와 순서는 매번 동일하게 유지한다.

## 제목 검증 기준

- 최신 이슈, 연예인, 시세, 추천, 계급도는 반드시 웹에서 확인한다.
- 제목이 앞뒤가 맞지 않을 가능성이 있으면 웹 검색과 기존 제목 자산을 비교한다.
- 같은 키워드 반복 발행을 피하고 주변 키워드로 확장한다.

## 2026-05-28 성과 학습

- 2026년 남자/여자 결혼 예물시계 추천 글은 홈판/추천형 성과가 가장 강했다. `2026년` + `남자/여자` + `결혼/예물` + `추천` + `TOP` 구조가 넓은 독자층을 만든다.
- 예물 주제를 반복하기보다 계급도, 가격대 추천, 브랜드 비교, 연예인 착용, 남자/여자 실착 기준으로 구조를 확장한다.
- `보증서 없는 롤렉스 데이저스트 매입 가격 많이 떨어질까?`는 조건 불안형 검색/전환 주제로 강했다.
- 지역+모델번호+매입 후기+가격이 달라지는 이유 구조는 검색형으로 안정적이다.
- 단순 브랜드/서비스 소개형 제목은 조회수가 약하다. 제목에는 고객 불안, 비교 기준, 가격이 달라지는 이유가 들어가야 한다.
- 과거 제목의 `감가 없이`, `최고가` 표현은 성과가 있어도 향후에는 단정 표현으로 쓰지 않는다. 감가 기준과 실물 확인 조건으로 바꾼다.
