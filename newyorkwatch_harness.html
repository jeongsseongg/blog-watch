  • CTA 이미지 바로 위 문단에서 전화 상담, 홈페이지 절차 확인, 카카오채널 사진 견적 세 행동을 텍스트로 한 번 더 설명한다.

발행 규칙

  • 기존 글 수정은 우선순위가 낮다. 새 콘텐츠로 상위노출과 AI 브리핑 점유를 노린다.
  • 같은 키워드 반복 발행을 피한다.
  • 명품시계매입 시세 노출 글이 이미 있으면 주변 키워드로 확장한다.
  • 네이버블로그 본문은 사진 설명과 CTA 설명을 제외하고 2500~3000자를 목표로 한다.
  • 제목, 이미지 위치 설명, 이미지 안 텍스트 제안, 사진 설명/대체텍스트, CTA 설명은 본문 글자 수에서 제외한다.
  • 매입후기와 실사 기반 포토형 글은 사진 → 핵심 텍스트 → 사진 → 핵심 텍스트 구조를 우선한다.
  • 목적은 글자 수를 줄이는 것이 아니라 이미지와 글을 조합해 읽기 쉽게 만드는 것이다. 가격 비교, 감정 기준, 감가 사유, 최종 매입 결과처럼 설득이 필요한 구간은 몇 줄이든 힘 있게 쓴다.
  • 사진은 6~8장을 사용한다.
  • 표는 이미지로 제작 가능하다.
  • 첫 300자 안에 결론을 넣는다.
  • 제목, 사진, 짧은 캡션은 중앙정렬 가능하나 정보형 본문은 왼쪽정렬을 권장한다.
  • 앞으로 포스팅 완성본을 줄 때는 이미지 생성 프롬프트까지 함께 제공한다.
  • 이미지 프롬프트는 기본 이미지 9장 + 하단 CTA 3장, 총 12개를 기준으로 한다.
  • 이미지 프롬프트는 사용자가 한 번에 복사하기 쉽게 마크다운 표나 여러 코드블록으로 쪼개지 말고 번호형 일반 텍스트로 제공한다.
  • 각 이미지 항목에는 후편집 텍스트, 프롬프트, 사진 설명/대체텍스트를 포함한다.
  • 하단 CTA 3장은 항상 1 전화, 2 홈페이지, 3 카카오채널 순서로 제공하고, 고정 문구는 `010-6293-6668 클릭 하시면 전화연결됩니다.`, `매입절차 및 최고가 약속`, `지금 사진보내고 가격확인`을 사용한다.
  • 짧고 큰 한글 핵심 문구는 이미지 안에 넣는다. 한 이미지에는 하나의 핵심 문구만 넣고 긴 설명/표/복잡한 가격표/작은 UI 텍스트는 넣지 않는다.
  • 글씨가 깨지지 않게 render crisp Korean Hangul typography, large clear Korean headline, no broken Korean text를 반드시 포함한다.

이미지 텍스트 규칙

  • 사용자가 보내는 실사 사진은 그대로 블로그에 붙이는 원본이 아니라 모델, 색상, 상태, 사용감, 버클/브레이슬릿 디테일을 읽는 참고 자료다.
  • 실사 사진 기반 원고는 실사 참고 → 핵심 장면 선별 → 이미지 안 텍스트 확정 → 실사감 있는 이미지 생성 프롬프트 → 본문 순서로 만든다.
  • 원고 작성 시 각 이미지마다 이미지 안 텍스트, 생성 프롬프트, 사진 설명/대체텍스트, 본문에서 맡는 역할을 함께 제안한다.
  • 본문 핵심 문장은 이미지 안에만 넣지 않는다. 중요한 내용은 본문 텍스트로도 쓴다.
  • 카카오톡이나 문자 상담 이미지는 실제 고객 상담처럼 만든다. 고객이 시계 사진을 보내고, 뉴욕워치가 1차 견적, 방문 안내, 입금 완료를 답하는 맥락이 보여야 한다.
  • 이미지 생성은 핵심 이미지 중심으로만 한다. 썸네일, 상담 대화, 구성품 확인, 감정 디테일, 비교 견적, 최종 매입, CTA 정도만 만들고 불필요한 장식 컷을 늘리지 않는다.
  • 이미지는 사진 설명/대체 텍스트에 핵심 키워드를 자연스럽게 넣는다.
  • 썸네일 이미지는 텍스트를 넣되 6~10단어 이내로 크게 쓴다.
  • 표 이미지는 사용 가능하나 표 아래에 결론을 2~3문장으로 다시 요약한다.
  • 일반 본문 사진에는 과한 텍스트를 넣지 않고 짧은 라벨만 사용한다.
  • 이미지 안 텍스트로 키워드를 반복 도배하지 않는다.
  • 원고 작성 시 각 이미지 위치마다 이미지 안 텍스트, 사진 묘사, 사진 설명/대체텍스트를 함께 제안한다.

이미지 로고 규칙

  • 사진마다 뉴욕워치 로고가 보여도 노출 자체에 불리하다고 보지는 않는다.
  • 로고가 피사체를 가리거나 광고 배너처럼 보이면 신뢰와 이미지 이해를 방해한다.
  • 대표 썸네일, 매장 사진, 상담 사진에는 로고 노출이 유리하다.
  • 파손 부위, 각인, 버클, 다이얼, 브레이슬릿 사진에는 로고를 작게 넣거나 배경에만 둔다.
  • AI가 이미지를 이해하는 핵심은 로고보다 피사체, 주변 문맥, 파일명, 대체 텍스트, 사진 설명이다.
  • 파일명과 사진 설명은 이미지 내용을 설명하게 작성한다.

이미지 생성 프롬프트 규칙

  • 이미지 생성 프롬프트는 원고만큼 신경 써서 작성한다. 저품질 광고 이미지는 브랜드 신뢰를 깎는다.
  • 절대 금지: 중국식 저가 광고 느낌, 어둡고 탁한 톤, 과한 골드/브라운/검정 그림자, 깨지는 한글, 어울리지 않는 SVG 이모지, 랜덤 아이콘, 과한 렌즈 플레어, 싸구려 3D, 과장된 광택, 복잡한 배경.
  • 절대 금지: AI 생성 단계에서 전화번호, 긴 한글 문장, 표, 가격, CTA 버튼, 작은 UI 텍스트를 모두 넣는 구성. 이런 구성은 글씨가 깨지고 중국식 저가 광고처럼 보인다.
  • 실사 참고 사진은 이미지 생성의 근거로만 사용한다. 그대로 복제하지 않고 감정 포인트, 색상, 상태, 디테일을 고급 광고 이미지로 재구성한다.
  • 카카오톡이나 문자 상담 이미지는 실제 고객 상담처럼 만든다. 고객이 시계 사진을 보내고, 뉴욕워치가 1차 견적, 방문 안내, 입금 완료를 답하는 장면이 핵심이다.
  • 이미지 생성은 핵심 이미지 중심으로만 한다. 썸네일, 상담 대화, 구성품 확인, 감정 디테일, 비교 견적, 최종 매입, CTA 정도만 만들고 불필요한 장식 컷을 늘리지 않는다.
  • 매번 다른 장면과 구도를 쓴다. 같은 스마트폰 목업, 같은 체크박스 카드, 같은 상담 테이블을 반복하지 않는다.
  • 실패 패턴으로 확정: 검정 배경, 골드 그라데이션 큰 글씨, 어두운 쇼룸, 과한 명품 조명, 두꺼운 한글 타이포, 장식 아이콘 조합. 이 조합은 중국식/저가 광고처럼 보이므로 금지한다.
  • 실패 패턴으로 확정: 화이트/블루만 과하게 쓰면서 스마트폰 목업, 빈 체크박스, 라운드 카드만 반복되는 이미지. 명품시계 전문성과 제품 차별성이 약하다.
  • 실패 패턴으로 확정: 모든 이미지가 같은 클립보드/카드 UI 틀로 반복되는 것. 네이버 이미지탭에서 한 장 한 장의 클릭 이유가 사라진다.
  • 권장 톤: 요즘 한국 광고/마케팅 회사가 만든 PPT형 광고 이미지, 밝은 배경, 선명한 브랜드 컬러 포인트, 큰 타이포 영역, 충분한 여백, 깔끔한 그리드, 고급 제품 사진, 전문적인 앱/카드 UI 목업, 명확한 CTA 버튼.
  • 권장 색감: 화이트, 라이트 그레이, 밝은 베이지, 깨끗한 네이비, 뉴욕워치 블루 포인트. 골드는 작은 포인트로만 사용한다.
  • 명품시계 이미지는 과하게 어둡고 시네마틱하게 만들지 않는다. 고급스럽되 밝고 깨끗하게, 상품과 감정 포인트가 잘 보이게 만든다.
  • 한글 텍스트는 생성형 이미지에서 깨질 위험이 크지만, 네이버 이미지탭 클릭을 위해 짧고 큰 핵심 문구는 이미지 안에 넣는다.
  • AI 이미지 안 텍스트는 2~6단어의 큰 문구만 사용한다. 작은 설명문, 긴 문장, 전화번호, 가격, 복잡한 표 텍스트는 이미지 생성 단계에서 넣지 않는다.
  • 프롬프트에는 render crisp Korean Hangul typography, large clear Korean headline, no broken Korean text, no random emoji icons, no cheap Chinese advertisement style, no dark black and gold luxury ad, no muddy brown tone, clean modern Korean marketing design, bright premium layout, professional presentation slide style을 포함한다.
  • 광고 이미지에는 아이콘보다 실제 시계, 감정 장면, 상담 장면, 앱/홈페이지 UI 목업, 체크리스트 카드 같은 오브젝트를 우선한다.
  • 계급도 이미지는 단순 앱 UI보다 정보 비교성이 우선이다. 등급표, 브랜드 묶음, 목적별 추천, 감정 기준, 보유 재고 비교가 한눈에 보여야 한다.
  • 밝게 만들되 너무 sterile하지 않게 뉴욕워치 실제 매장 질감, 시계 디테일, 감정사 손, 쇼케이스, 상담 장면을 섞는다.
  • 레퍼런스 방향은 큰 헤드라인, 선명한 컬러, 단순한 오브젝트, 하단 CTA가 분명한 구성이다.
  • 최종 이미지 프롬프트에는 금지 스타일, 권장 색감, 구도, 텍스트 처리 방식, 오브젝트, 브랜드 톤을 모두 명시한다.

뉴욕워치 이미지 비주얼 시스템

  • 레퍼런스는 참고용이다. 그대로 복제하지 않고 색감, 여백, 정보 구조, CTA 배치, 제품 노출 방식만 참고한다.
  • 뉴욕워치 로고 기준 색상은 밝은 블루와 화이트다. 이미지 전체를 검정/골드로 끌고 가지 않는다.
  • 매장 분위기는 따뜻한 베이지, 스톤, 우드, 은은한 조명이다. 광고 이미지에서는 이 톤을 밝고 깨끗하게 재해석한다.
  • 기본 광고 색상은 화이트, 라이트 그레이, 페일 블루, 뉴욕워치 블루, 딥 네이비 텍스트다.
  • 보조 색상은 밝은 베이지, 실버, 아주 작은 골드 포인트다.

AI 이미지 제작 분업 원칙

생성형 AI는 촬영 배경을 만들고, 정확한 한글 문구와 표는 디자인 레이어로 통제한다.
구분 판단 실행 방식
AI가 잘하는 영역 실사감 있는 상황, 구도, 시계 진열, 매장 무드, 상담 대화, 감정 장면 짧은 핵심 문구와 실제 상황이 함께 보이는 이미지로 생성한다.
AI가 약한 영역 긴 한글 문구, 표, 다중 텍스트 레이어, 브랜드명, 숫자, CTA Canva, Photoshop, PPT에서 직접 얹는다.
표 이미지 한글, 정렬, 브랜드명, 숫자 정확성이 핵심이라 AI 생성 실패율이 높다. 절대 AI에게 완성 표를 맡기지 않고 디자인 레이어로 제작한다.
브랜드 로고 AI가 까르띠에, IWC, 오메가, 롤렉스 로고를 임의로 왜곡하거나 누락할 수 있다. 사용자가 제공한 실제 뉴욕워치 로고와 필요한 공식 로고 자산만 사용한다.
  • 9장 이상 연속 이미지처럼 각 이미지마다 정확한 문구와 구조가 지정된 작업은 AI 완성 이미지 방식으로 진행하지 않는다.
  • AI 이미지에는 짧고 큰 핵심 문구만 넣는다. 긴 설명, 표, 복잡한 숫자, 작은 UI 텍스트는 넣지 않는다.
  • 핵심 이미지는 실제 상황처럼 만든다. 사진 전송, 감정 확인, 비교 견적, 최종 매입, CTA 장면이 우선이다.
  • CTA 이미지는 고정 문구 010-6293-6668 클릭 하시면 전화연결됩니다., 매입절차 및 최고가 약속, 지금 사진보내고 가격확인를 그대로 쓰되, 배경 장면과 구도는 매번 다르게 만든다.
  • 권장 폰트는 Pretendard, Noto Sans KR, SUIT 계열이다.
  • 권장 색상은 네이비, 화이트, 밝은 베이지, 뉴욕워치 블루 포인트다.
  • 이미지 작업 요청을 받으면 배경 이미지 생성안과 별도로 문구, 배치안, 폰트 크기, 컬러, 위치를 제공한다.

매입후기 작성 예시

매입후기는 정보 설명문이 아니라 실제 사건처럼 써야 한다. 고객이 왜 왔고, 무엇을 걱정했고, 뉴욕워치가 무엇을 다르게 봤고, 최종 결과가 어떻게 끝났는지가 보여야 한다.

  • 매입후기 참고 예시는 newyorkwatch_blog_writing_examples.md에 저장한다.
  • 기본 흐름은 고객 상황 → 기존 견적 또는 걱정한 감가 사유 → 방문/사진 상담 과정 → 실제 확인한 구성품과 상태 → 타 업체 판단과 뉴욕워치 판단 차이 → 최종 매입 결과 → 왜 뉴욕워치인지 → CTA 순서다.
  • 화면 배치는 사진 → 핵심 텍스트 → 사진 → 핵심 텍스트 구조를 기본으로 한다. 짧게 넘길 구간은 짧게, 가격·감정·비교처럼 설득이 필요한 구간은 충분히 설명한다.
  • 핵심 컷은 썸네일, 상담/카톡 상황, 시계 전체컷, 구성품/보증서/서비스카드, 감정 디테일, 비교 견적, 최종 매입, CTA 순서로 선별한다.
  • 모델 설명보다 고객이 왜 왔고 무엇을 걱정했는지 먼저 보여준다.
  • 감정 기준은 일반론으로 반복하지 말고 보증서, 서비스 카드, 행택, 레드 왁스씰, 여분 링크, 다이얼 상태, 브레이슬릿 늘어짐, 케이스백 산화처럼 해당 시계의 구체적 증거와 연결한다.
  • 여러 업체 비교, 타 업체 감가 사유, 최종 매입가, 감가 없이, 100만 원 더, 정상 매입 같은 결과 표현은 실제 사실일 때만 사용한다.
  • 왜 뉴욕워치일까요? 파트는 단순 홍보가 아니라 이번 사례에서 뉴욕워치 판단이 달랐던 이유를 다시 정리하는 용도로 쓴다.
  • 거래 절차는 사진 견적 → 방문/출장/택배 선택 → 실물 검수 → 확정 견적 → 즉시 입금 또는 현금 지급 흐름으로 설명할 수 있다.
  • 사진에서 이미 확인된 생활기스나 작은 사용감은 1차 견적에 반영한다는 원칙을 설명한다. 단, 실제 사례에서만 1차 견적과 확정 견적 동일이라고 쓴다.
  • 감가사항은 무브먼트 손상, 침수, 녹, 내부 부품 파손, 수리 불가 손상, 가품 판정처럼 사진으로 확인하기 어려운 문제를 구체적으로 고지한다.

AI 브리핑 작성 원칙

  • AI 브리핑은 직접 통제할 수 없다. 글의 목표는 네이버가 출처로 선택하기 쉬운 검색 의도 대응 문서가 되는 것이다.
  • 긴 감성글보다 결론, 기준, 조건, 실제 사례가 분리된 문장을 우선한다.
  • 제목 앞부분에는 실제 검색 의도와 핵심 키워드를 배치한다.
  • 첫 300자 안에 가능 여부, 판단 기준, 필요 사진, 최종가 확정 조건을 넣는다.
  • 핵심 의심이 있는 구간은 질문형 소제목을 사용하되, 모든 소제목을 기계적으로 질문형으로 만들지는 않는다.
  • 본문은 결론 → 근거 → 실제 사례 또는 판단 기준 → CTA 순서로 전개한다.
  • 표, 체크리스트, 요약 문단을 사용하되 핵심 결론은 본문 텍스트로 다시 설명한다.
  • 단순 후기보다 감정 기준, 가격이 달라진 이유, 거래 절차가 포함된 실제 사례 결합형을 우선한다.
  • 시세와 매입가는 고정 비율로 단정하지 않고 모델, 상태, 구성품, 국내외 수요, 재판매 가능성 기준으로 설명한다.
